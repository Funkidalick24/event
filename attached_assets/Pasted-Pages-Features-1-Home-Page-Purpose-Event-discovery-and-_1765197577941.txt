Pages & Features
1. Home Page (/)
Purpose: Event discovery and listing
Features:
Displays all available events in a clean list format
Shows event title, date, and description
"Register" button for each event (links to registration form with event pre-selected)
Real-time data loading from database
Responsive design
2. Registration Form (/register)
Purpose: User registration for events
Features:
Interactive form with live validation
Required fields: Full Name, Email
Event selection via URL parameter (from home page)
Client-side validation (HTML5 + React state)
Form submission saves to database
Automatic redirect to confirmation page after successful registration
3. Registration Confirmation (/registrations/[id])
Purpose: Display registration details
Features:
Dynamic routing based on registration ID
Shows user's registration information
Displays associated event details
Confirmation of successful registration
Unique registration ID for reference

-- Registrations Table
CREATE TABLE Registration (
  id SERIAL PRIMARY KEY,
  eventId INTEGER REFERENCES Event(id),
  fullName VARCHAR NOT NULL,
  email VARCHAR NOT NULL
);
